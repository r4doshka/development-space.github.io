Even if you did initial customer research and have collected some people willing to use your future product. You’ll only know people ready to use product after they actually started using it. Test idea with MVP and get feedback - that’s important.
MVP allows you to test if your idea is technically realisable. There is no sense to make fully working product, until you’re sure that all the features you plan to build are technically possible. With MVP you can test Scalability, 3rd party integrations and algorithm realisation.
Mobile apps and browsers have certain limitations that we should take into account. If your key feature is based on working around those limitation, you should start from it and test if it’s technically realisable.

What’s next?
1
Product planning stage
You work with project coordinator and discuss customer profile, business model and organization of offline processes - then, we’ll set project goals and identify main use cases.
Result
  • Customer journey map and customer profile
  • Product mindmap
  • Clear project requirements
2
Designing and Wireframing stage
You work with Project manager and designer. Define the interface structure, choose color palette and fonts for the service.
Result
  • Complete design of the whole website
  • Clickable prototype (without functionality)
  • Website logo
3
Development stage
Design conversion into html and css. Implementation of the functional part of the site. Staging server. Going live.
Result
  • Live web service
  • Set user analytic tools
  • Feedback gathering tools
4
Iterative development stage
Study customers’ feedback about MVP and decide which way to go. Integrate new functionality iteration by iteration. Growing from mini-service into fully functional online business.
Result
  • Growing interface
  • More analytics
  • New insights about the product
